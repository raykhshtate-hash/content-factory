# Roadmap: Content Factory — Sprint 1+1.5

## Overview

This milestone evolves Content Factory from a functional render pipeline into a creative production tool. The roadmap progresses from production stability (retry, billing, versioning) through quality tooling + basic feedback UI (regression harness, Gemini polish, approve/redo buttons), creative effects (animated text, SFX), structured creative planning (ScenePlan), advanced feedback-driven iteration (selective re-render, classification trace), and finally advanced layouts (PiP, split screen). Core features (feedback loop, ScenePlan, SFX, text animations) must deploy; edge features (PiP, split screen) can slip.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: Stability & Observability** - Error recovery, billing tracking, and prompt versioning for confident iteration
- [ ] **Phase 2: Quality Harness, Gemini Polish & Basic Feedback** - Regression testing, Gemini prompt improvements, and approve/redo UI as quick wins
- [ ] **Phase 3: Animated Text & SFX** - Motion text overlays, B-roll text, and transition-matched sound effects
- [ ] **Phase 4: ScenePlan & Creative Pipeline** - Structured briefs replacing scenario_text, capability registry, directed stickers
- [ ] **Phase 5: Advanced Feedback & Iteration** - Selective re-render routing and classification trace logging powered by ScenePlan
- [ ] **Phase 6: Advanced Layouts** - PiP-lite overlay and split screen content mode

## Phase Details

### Phase 1: Stability & Observability
**Goal**: Pipeline failures are recoverable and every render has cost + version traceability
**Depends on**: Nothing (first phase)
**Requirements**: STAB-01, STAB-02, STAB-03, STAB-04, BILL-01, BILL-02, BILL-03, QUAL-03
**Success Criteria** (what must be TRUE):
  1. A Creatomate 500 error triggers automatic retry and the user eventually receives their render (or a clear failure message)
  2. A Gemini safety filter rejection triggers retry with adjusted parameters instead of pipeline failure
  3. User can tap "Retry render" in Telegram to re-submit a failed render without restarting the full pipeline
  4. After each render, the bot sends a cost breakdown (Whisper, Gemini, Claude, Creatomate) in Telegram
  5. Every render log includes gemini_prompt_version and director_prompt_version for traceability

### Phase 2: Quality Harness, Gemini Polish & Basic Feedback
**Goal**: Gemini prompt changes are tested against fixtures before deploy, clip selection quality improves, and users can approve or request redo on delivered renders
**Depends on**: Phase 1 (prompt versioning must exist for regression tracking)
**Requirements**: QUAL-01, QUAL-02, QUAL-04, FEED-01, FEED-02
**Success Criteria** (what must be TRUE):
  1. Running `python scripts/gemini_regression.py` executes 8-12 fixture cases and reports pass/fail with clip count, duration, dedup, and diversity assertions
  2. Gemini clip selection shows measurably fewer duplicate sources and less linear timestamp ordering on existing test videos
  3. Regression harness catches prompt regressions before deploy (a deliberately broken prompt fails assertions)
  4. After render delivery, Telegram shows "Approve" and "Redo" buttons
  5. User taps "Redo", types feedback, and Haiku classifies it as gemini_instruction and/or director_instruction (routing only, no selective re-render yet)

### Phase 3: Animated Text & SFX
**Goal**: Reels have motion text overlays instead of static popups, unmatched B-roll gets contextual text, and transitions have matched sound effects
**Depends on**: Phase 1 (stability ensures safe iteration on render JSON)
**Requirements**: VFX-01, VFX-02, VFX-03, VFX-04, AUD-01, AUD-02, AUD-03, AUD-04
**Success Criteria** (what must be TRUE):
  1. Text overlays in rendered reels animate (fade, slide-up, typewriter, or pop) instead of appearing/disappearing instantly
  2. Long text automatically falls back to fade animation with duration caps (no broken layouts)
  3. Unmatched B-roll clips display a short humorous or question text overlay (3-5 words) generated by Gemini
  4. Transitions in rendered reels play a matching sound effect (whoosh for wipe, swoosh for slide, etc.) on a dedicated audio track

### Phase 4: ScenePlan & Creative Pipeline
**Goal**: Users describe a creative vision in free text and get a structured plan that drives clip selection, sticker placement, and effects
**Depends on**: Phase 3 (text animations and SFX exist as capabilities for the registry)
**Requirements**: PIPE-01, PIPE-02, PIPE-03, PIPE-04, PIPE-05, PIPE-06, PIPE-07
**Success Criteria** (what must be TRUE):
  1. The pipeline uses a ScenePlan JSON (with schema_version) instead of scenario_text for all render modes
  2. User can type a free-form brief in Telegram and receive a structured ScenePlan preview before render
  3. Sticker quantity and placement are driven by ScenePlan instructions (not random auto-fill), with auto-fill only on permissive briefs
  4. The capability registry (AVAILABLE_CAPABILITIES) is injected into the planner prompt, so ScenePlan only references effects the pipeline can render
  5. ScenePlan moments[] flow works end-to-end: Claude creates skeleton, Gemini fills clips

### Phase 5: Advanced Feedback & Iteration
**Goal**: Feedback-driven re-renders target only the affected pipeline stage, with full classification traceability
**Depends on**: Phase 4 (ScenePlan enables structured re-render with targeted instructions), Phase 2 (basic feedback UI and Haiku classification already working)
**Requirements**: FEED-03, FEED-04
**Success Criteria** (what must be TRUE):

  2. Classification trace is logged to Supabase (feedback_text, classified_type, affected_dimensions, acceptance) for every redo interaction

### Phase 6: Advanced Layouts
**Goal**: Reels can include picture-in-picture overlays and side-by-side split screen compositions
**Depends on**: Phase 4 (PiP is brief-driven via ScenePlan)
**Requirements**: VFX-05, VFX-06, VFX-07, VFX-08
**Success Criteria** (what must be TRUE):

  3. Split screen renders as a dedicated content mode with top/bottom 1080x960 layout and text labels
  4. Split screen uses a specialized Gemini prompt to select paired clips that make sense together
